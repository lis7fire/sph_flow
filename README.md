# 视频号流速监控系统（Python 版）

本项目是本地运行的视频号数据采集与流速监控工具。启动后通过浏览器访问本地页面，配置视频号后台 Cookie，选择要采集的视频，程序会按周期在后台采集数据并展示趋势。

## 功能

- 本地 Web 页面管理采集配置和账号配置。
- 支持多账号，每个视频号账号单独保存一组 Cookie。
- 添加或编辑账号时使用弹窗填写 Cookie，并自动获取账号名称；获取不到时可以手动填写。
- 采集设置自动保存，不再需要统一的“保存设置”按钮。
- 每个账号可以单独选择视频、暂停采集、恢复采集、删除账号。
- 采集前读取近期视频列表，勾选本次要采集的视频。
- 后台按周期采集已选择的视频数据。
- 展示最近采集快照、周期汇总、指标趋势折线图。
- 支持导出 Excel。

## 运行

```powershell
D:\python_project\sph_flow\.venv\Scripts\python.exe D:\python_project\sph_flow\main.py
```

启动后访问：

```text
http://127.0.0.1:8765
```

默认监听地址和端口为 `127.0.0.1:8765`。

## 账号配置

1. 打开页面后点击“添加账号”。
2. 粘贴该视频号后台请求中的完整 `Cookie`。
3. 点击“获取账号名”，程序会请求微信后台账号信息接口。
4. 如果账号名获取成功，账号名称和账号 ID 会自动填入。
5. 如果获取不到账号名，可以手动填写“账号名称”后保存。

字段来源：

- “账号名称”来自账号信息响应中的 `finderUser.nickname`。
- “账号 ID（可选）”来自响应中的 `finderUsername`。
- 不使用 `userAttr.nickname` 作为账号名称，也不使用 `userAttr.finderUsername` 作为账号 ID。

删除账号会立即保存，不需要再点额外的保存按钮。

## 选择视频与采集

- 点击账号后的“选择视频”会读取该账号近期视频。
- 只要微信接口成功响应且业务码为 `0`，就认为读取成功。
- 如果账号没有视频，会显示“官网请求成功，没发现数据。”，不会当作错误。
- 如果微信返回非 `0` 业务码，会显示错误，例如 Cookie 过期、账号不匹配或接口权限问题。
- 错误里的接口名只显示最后一级名称，例如 `post_list`，不会显示完整路径。

选中视频后，程序会按“采集间隔(分钟)”自动后台采集。账号也可以单独暂停或恢复采集。

## 数据与日志

运行数据保存在项目 `data` 目录下：

- `data/sph_flow.sqlite3`：配置、视频、采集快照。
- `data/logs/sph_flow.log`：运行日志和接口请求日志。

页面右上角“导出 Excel”会导出已采集的快照数据。

## 常见提示

- `官网请求成功，没发现数据。`：Cookie 有响应，业务码成功，但该账号近期没有可读取的视频。
- `连接成功，但微信返回业务码 ...`：HTTP/网络已连通，但微信业务层返回失败，常见原因是 Cookie 过期、Cookie 不属于该视频号、账号无权限或请求过于频繁。
- `接口返回不是合法 JSON`：通常是登录态失效，微信返回了 HTML 或跳转页，需要重新复制 Cookie。
